列举一下大作业文档中的要求：

### 基本要求

对于具体任务部分，应该都可以单独创建一个 App 进行测试。

+ 具有基本体素（⽴⽅体、球、圆柱、圆锥、多⾯棱柱、多⾯棱台）的建模表达能⼒；

    应该是指能自行建模简单几何体。如果是这个意思那没啥要做的，实现模型 `.obj` 等格式导入之后直接建模即可；

    不过可能具体意思要和助教问一下

    **具体任务**：简单建模即可，没啥要做的。

+ 具有基本三维⽹格导⼊导出功能（建议OBJ格式）：

    + 需要了解OBJ格式，完成基本OBJ模型的读⼊和导出，不要求处理⽂件中的纹理和材质信息。（不能⽤⼩作业⾥的tiny obj）

    OBJ 格式基本包括几何体顶点 v、贴图坐标点 vt、顶点法线 vn、面 f，以及其他标注模型信息的字段等。应该是要自行实现将文件内容导入到 `Model` 对象中。

    **具体任务**：对 `Model` 类的构造函数进行重构，将目前的 `tinyobj` 替换为自己的实现。

+ 具有基本材质、纹理的显⽰和编辑能⼒；

    应该指 Project 6，可能需要确认一下是否能使用 CG-Project 内的 utils。

    **具体任务**：如果自己实现，也是简单重写一下 `Texture` 等类。可能要稍微注意一下复杂模型的多个材质和网格的映射问题。

+ 具有基本⼏何变换功能（旋转、平移、缩放等）；

    应该指 Project 2。

    **具体任务**：感觉没啥好说的，[Learning OpenGL-变换](https://learnopengl-cn.github.io/01%20Getting%20started/07%20Transformations/#glm)

+ 具有基本的光源照明要求，并实现基本的光源编辑（如调整光源的位置，光强等参数）；

    指 Project 5 即没有涉及的更多内容。例如，如果光源很多，如何在着色器里配置可变数量的光源；pbrt 等高级光照效果。

    **具体任务**：对于基本部分即 Project5。

+ 能对建模后场景进⾏漫游如Zoom In/Out, Pan, Orbit, Zoom To Fit等观察功能。

    指 Project 3。需要注意的是 project 3 只实现了漫游，此处还需要摄像机和物体交互等。不过感觉给 `Camera` 类加功能就 ok

    **具体任务**：给 `Camera` 类增加不同的相机运动模式，例如支持通过用户输入调整截头体、支持围绕一个点进行 orbit 模式等。对于 Zoom To Fit，可能还包含相机从原位置移动到物体位置的平滑动画。

+ 能够提供动画播放功能（多帧数据连续绘制），能够提供屏幕截取/保存功能。

    + 多帧连续绘制指读取连续的OBJ⽂件（或其他格式的模型⽂件），对其⽹格进⾏多次的绘制；
    
    + 简单的平移、旋转、缩放等不能视为动画播放。
    
    指的是模型形状变化，例如 3D 火焰播放等；而对于人物头发、裙摆等物理效果，在 EX 中涉及。

    屏幕截取/保存可能要研究一下 OpenGL 里帧缓冲内的数据该怎么获取。

    **具体任务**：多帧连续读取 OBJ 文件并绘制；帧缓冲读取

### EX

+ 构建了基于此引擎的三维游戏，具有可玩性

    + 要求提出核⼼玩法并加以实现，对于⾮核⼼玩法的部分可从简

    + 实现场景编辑器，可以在场景中添加、删除、复制、编辑物体，可以保存和加载场景，以实现的正确性、易⽤性、功能性为评价标准

    感觉还应该再讨论一下具体什么玩法。

    **具体任务**：如果是射击游戏，那么需要实现弹道检查、血量检测、人物死亡等。

+ 实现实时碰撞检测
  
    + 碰撞检测必须是基于空间⼏何的碰撞检测，仅使⽤平⾯位置进⾏的边界判定不视为碰撞检测
    
    + 根据各种模型的形状，包围盒需要有区分，你总不能⽅块和兔⼦都当球来处理吧
    
    + 离散碰撞检测就可以了，有能⼒的可以试试连续碰撞检测

    哦哦的耶，计算几何

    **具体任务**：实现碰撞检测，能够报告两个物体发生了碰撞等。更进一步可能需要实现碰撞了之后两者的速度怎么变化，巨大物理

+ ⾼级光照和阴影效果
  
    + 可选项：实现 Blinn-Phong 模型、实时阴影、透明度和反射效果、辉光/泛光（Glow/Bloom）、相机散焦模糊（Caustic效果）、PBR、HDR、位移纹理（Displacement Map）、环境光遮蔽、辐射度、光线追踪等
    
    + 可以参考unity/unreal等游戏引擎的渲染实现
    
    + 以最终实现的渲染效果作为评价标准

    感觉是一个大头任务且需要更进一步细分，不过各种效果都是可选项，应该可以直接对可选项中的子模块领任务。

    需要注意的是最好不要一股脑全上，可以根据最终想要达到的风格需要什么效果进行选择。

    **具体任务**：对于不同的模型实现着色器和外部使用部分。

+ 实现三维场景中的中⽂⽂字渲染

    + 以实现效果作为评价标准（包括但不限于在场景中动态⽣成、显⽰和编辑⽂字；⽀持设置⽂字的样式（字体、⼤⼩、颜⾊等）和对⻬⽅式（水平、垂直对⻬等），以及在⽂字上应⽤纹理和材质效果）

    主要内容是通过文字生成模型或者材质参与渲染。

    **具体任务**：研究一下对于文本框动态生成文字模型的技术。

+ 具有⼀定的对象表达能⼒

    + 例如，能够表达⻔、玻璃，泥地、铁板等对象，要求具有实际功能上的不同，如CS:GO的模型中：关闭的⻔不可通⾏，可以按E键打开，打开后可通⾏；玻璃不可通⾏，但是可打碎；⼈物在泥地和铁板上⾛动时的声⾳反馈不同等等

    需要实现模型行为，例如继承 `Model` 实现 `Door` 子类，在交互时可以旋转 90° 等，并存储开关状态；或者基于物理转轴实现，但一般游戏的思路应该都是前者。

    **具体任务**：在 `Model` 中实现不同的模型行为。可能需要注意继承关系，例如 `CollideModel, TransparentModel, Door, Player` 等相互关系。

+ ⾼级动画效果

    + 例如， 实现具有⼀定复杂度的关键帧动画或路径动画（如⻉塞尔曲线路径）；实现基于物理的动画效果，如布料模拟或流体模拟

    **具体任务**：对于贝塞尔曲线等刚体动画，可以完善 `Movement`，增加中间轨迹不同的类；对于物理模拟，需要继承 `Model` 类，对顶点实现弹簧等变换。

+ 基于GPU的加速：使⽤CUDA或OpenCL加速场景中的计算（如复杂模型的碰撞检测，BVH 加速结构等）

    cuda 领域大神

    BVH 感觉可以参考 `utils/bounding_box`

    **具体任务**：此处计算应该指非渲染管线部分，那其实是 trivial cuda。可以看一下具体实现时，哪一部分有大量并行可以优化。

有些任务还不太明确，下面的任务列表省流版会具体说明要干什么，会慢慢更新：

+ `Camera` 运动模式添加：改变截头体、实现环绕某物体中心坐标（维护一个中心 position）等。可以在 `Camera` 类中添加摄像机运动模式，根据不同模式改变各运动算法。
+ `obj` 模型导入+材质：重写 `Model` 类的构造函数，自己实现将文件中数据导入 `Model` 成员变量；对于材质部分，注意 `Vertex` 类里有材质坐标；不过对于法向部分，注意有顶点法向和面法向两种法向存储&计算方式，而 Vertex 中只提供了顶点法向。
+ 碰撞检测：由于和物理部分解耦，这一部分可以先在命令行输出各个物体是否相撞，并直接通过输入来改变各个物体的坐标；应该需要部分计算几何知识。
+ 物理效果：可以全局设置重力加速度 g 等。除了自由落体，还可以测试平抛运动、加上物理质量等。不过带旋转的刚体物理可能较为困难，可以先实现不带碰撞检测的反弹（指如果某一维坐标超过 boundary 则改变速度；支持任意角度的速度和 boundary 反弹）
+ 光照效果：可以实现多光源计算。注意阴影是 bonus 部分，可以先实现点光源、日光、环境光、聚光等 trivial 效果。
+ 模型动画：维护多帧模型进行滚动绘制。

注意都可以通过创建不同的 `App` 进行测试。

文件结构可以先按照自己的大致思路放，这个之后可以慢慢改。
